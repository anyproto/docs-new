# Collections

### What is a Collection?

A **Collection** is a manual grouping of Objects. You pick which Objects go in, and they stay until you remove them. Unlike Queries (which automatically pull in Objects matching certain criteria), Collections start empty and you build them up yourself.

The key difference: a Collection can hold Objects of different Types together. A project Collection might contain tasks, notes, meeting minutes, and reference documents — all in one place.

### Why it matters

Sometimes you need to group things that don't share a common Type or Property — they just belong together because of context. A Collection gives you a simple way to gather related Objects regardless of what Type they are.

### How it works

Collections work like a list you curate by hand. You add Objects to it, and they appear in the Collection view. You can use the same layout and view options available in Queries (Grid, List, Gallery, etc.).

{% hint style="warning" %}
The main difference between Queries and Collections is that Queries draw Objects from your entire graph based on filter criteria, whereas Collections start empty and you add Objects to them manually.
{% endhint %}

### Building a project Collection

Let's say you're working on a product launch and want to keep all related items in one place:

1. Create a new Collection from the sidebar using the **+** dropdown, or type `/collection` in the editor.
2. Give it a name — for example, "Product Launch".
3. Click **New** to create Objects directly inside the Collection. Try creating a Task ("Write announcement"), a Note ("Launch timeline"), and a Page ("Feature overview"). Notice how different Types live side by side.
4. To add existing Objects, hover over the arrow menu next to the New button and select **Existing object**. Search for any Object in your Channel and add it.
5. Use the layout and filter options to organize the view the way you prefer.
