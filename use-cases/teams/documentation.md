# Documentation

### What you'll build

A shared documentation hub for your team — a structured space where knowledge lives, stays up to date, and is easy for anyone to find.

{% hint style="info" %}Screenshot needed: overview of the documentation Channel from your team space{% endhint %}

### Concepts used

- [Channels](../../getting-started/vault-and-key/space.md) — a shared Channel for your team
- [Collaboration](../../getting-started/collaboration.md) — inviting team members with Editor access
- [Types](../../getting-started/types/README.md) — a "Doc" or "Article" Type for documentation pages
- [Properties](../../getting-started/types/relations.md) — for categorization (Team, Status, Last Updated)
- [Collections](../../getting-started/sets/collections.md) — to organize docs by topic or team
- [Sidebar](../../getting-started/customize-and-edit-the-sidebar.md) — for quick navigation to key docs

### Step-by-step setup

1. **Create a shared Channel.** Name it something clear like "Team Docs" or your team's name. Invite members with Editor access.
2. **Create a "Doc" Type** (or use the built-in Page Type). Add Properties:
   - **Select:** "Status" (Draft, Published, Needs Update)
   - **Select:** "Team" (Engineering, Design, Product, etc.)
   - **Date:** "Last Reviewed"
3. **Create a Template for documentation pages.** Include a standard structure: Overview, Details, Related Links. This keeps your docs consistent across the team.
4. **Organize with Collections.** Create Collections for major topics — "Onboarding", "Engineering Guides", "Design System", etc. Add relevant docs to each.
5. **Pin key Collections to the sidebar.** Everyone on the team will see them in the shared Channel's sidebar.
6. **Create a "Needs Update" Query.** Filter by Status: Needs Update. Pin it to the sidebar so outdated docs are always visible.

{% hint style="info" %}
**Tip:** Use inline links (@) to cross-reference between docs. Over time, this builds a web of interconnected knowledge that's easy to navigate.
{% endhint %}
